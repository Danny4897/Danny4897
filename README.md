# Hi, I'm Danny (Daniele) 👋

> I build the infrastructure that makes AI reliable in .NET.

I'm a C# engineer at the intersection of **functional programming** and **LLM integration**. I created **MonadicSharp** — an open-source ecosystem that brings Railway-Oriented Programming and production-grade AI reliability patterns to the .NET world.

I work with the **Anthropic Claude API**, **Model Context Protocol (MCP)**, and **.NET 8** to design systems where LLM failures are first-class citizens — not afterthoughts.

---

## 🔷 The MonadicSharp Ecosystem

A suite of composable libraries for building reliable, AI-native .NET applications:

| Package | What it does |
|---|---|
| [**MonadicSharp**](https://github.com/Danny4897/MonadicSharp) | Core — `Result<T>`, `Option<T>`, Railway-Oriented pipelines |
| [**MonadicSharp.AI**](https://github.com/Danny4897/MonadicSharp.AI) | LLM-first error handling — typed AI errors, retry, streaming, agent observability |
| [**monadic-sharp-mcp**](https://github.com/Danny4897/monadic-sharp-mcp) | MCP server — native Claude AI integration for code generation |
| [**MonadicSharp.Framework**](https://github.com/Danny4897/MonadicSharp.Framework) | Full-stack framework with clean architecture templates |
| [**MonadicSharp.DI**](https://github.com/Danny4897/MonadicSharp.DI) | Dependency injection integration |
| [**MonadicSharp.Azure**](https://github.com/Danny4897/MonadicSharp.Azure) | Azure-native extensions |
| [**MonadicSharp.Recovery**](https://github.com/Danny4897/MonadicSharp.Recovery) | Resilience patterns — circuit breaker, retry, fallback |

## 🧠 What I care about

- **Green Code** — minimizing wasted compute and LLM token consumption by design
- **Composability** — pipelines that fail explicitly, recover gracefully, and stay readable
- **AI tooling** — making LLMs a reliable dependency, not a liability
- **Developer experience** — `dotnet new monadic-api` should just work
